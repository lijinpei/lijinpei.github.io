Markdown 总结

Basic Syntax

heading <h1>,<h2>等
#, ##(井号后需空一格); 或===，---(一个vim小技巧,"C-v y p C-v r =")。
paragraph <p>
空一行。除非在list中，paragraph开头不要缩进。
line break <br>
显式添加html tag。
emphasis包括bold和italic
bold
**text**或__text__
italic
*text*或_text

既bold又italic: 三个星号或下划线。

blockquote
>。blockquote可以嵌套，可以包含其他markdown element（并不是所有element都可以使用）。
lists包括有序列表和无序列表
ordered lists'
num.。num必须以1开头，但不需要按顺序。
unordered lists
-，*，+。

要在list中添加其他元素，并在之后继续list，将元素缩进4个空格或者1个tab。

code block 代码块
四个空格或者1个tab缩进。在列表中需要8个空格或者2个tab缩进。
image
![名称](路径)
linking image(就是点击会跳转到链接的image)
[![名称](路径)](url)
代码<code> tag
`code`。要在back-tick中包含back-tick，需要用两个back-tick做引号。
horizontal rules
单独一行3个或以上***，---或___。
link
[link text](url "tooltip")
url
<url> <邮箱地址> url中的空格使用base64编码%20
reference-style link
  1. 第一部分 [text][label];
  2. 第二部分 [label]: url 描述;label不区分大小写
转义
\\。可以转义的字符参见这里
block-level html element

使用空行分割

不能在block-level html tag中使用markdown语法

extended syntax

扩展语法是与工具相关的

  • CommonMark
  • GitHub Flavored Markdown(GFM)
  • Markdown Extra
  • MuiltiMarkdown
  • R Markdown
tables

https://www.tablesgenerator.com/markdown_tables
