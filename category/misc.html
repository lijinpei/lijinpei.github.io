Markdown 总结

Basic Syntax

heading <h1>,<h2>等 :

paragraph <p> :

line break <br> :

emphasis包括bold和italic

bold:

italic:

blockquote : >

lists包括有序列表和无序列表

ordered lists

unordered lists

code block 代码块

四个空格或者1个tab缩进

在列表中需要8个空格或者2个tab缩进

image :![名称](路径)

linking image(就是点击会跳转到链接的image) :[![名称](路径)](url)

代码: ``

在代码中使用backtick : 双backtick escape

horizontal rules

link : [link text](url "tooltip")

url : <url> <邮箱地址>

url中的空格使用base64编码%20

reference-style link

第一部分 [text][label];第二部分 [label]: url 描述;label不区分大小写

转义 \\

使用空行分割block-level html element

不能在block-level html tag中使用markdown语法

extended syntax

扩展语法是与工具相关的

CommonMark GitHub Flavored Markdown(GFM) Markdown Extra MuiltiMarkdown R Markdown

    Sed 总结

    sed makes only one pass over the input.

    确定脚本，输入，输出

    '-'作为文件名表示标准输入(由sed解释)。

    -i 选项in-place 覆盖输入文件，可指定备份输入文件的后缀。

    默认打印所有被处理的输入，除了用d选项删除的。-n选项只打印p命令打印的文本。

    默认所有文件当成一个长的流。-s选项处理每个输入文件为单独的流。

    无-e, -f选项时第一个非选项参数为脚本;有-e, -f选项时，所有非参数选项均为输入文件。

    未指定输入文件时，从标准输入获取输入。

    -n
--quiet
--silent

    --debug

    -e script

    -f script-file
