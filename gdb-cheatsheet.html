5 暂停与继续

    info program 显示关于程序的信息

5.1 breakpoint watchpoint catchpoint

    breakpoint: 运行到某一行暂停
    watchpoint: 如果表达式的值改变了暂停
    catchpoint: 当某种事件发生时暂停（抛出C++异常，装载动态库，调用系统调用）

    三种"断点"统一编号。

5.1.1 设置断点

    break location 设置断点
            断点在指定的位置前(也就是说停下来的时候，指定的位置包含的指令还没有执行)。
    break 在当前frame下一条要执行的指令处设置断点。
            若当前frame不是innermost，效果相当于当执行返回到当前frame时暂停。类比finish。
            若当前frame是innermost，在当前指令上下断点（也就是下次执行到当前指令时）。
            gdb在恢复执行时（continue），在执行一条指令之前会忽视断点（否则无法越过一个断点，除非临时disable）。
    break ... if cond条件断点
    tbreak args 单次断点
    hbreak args 设置硬件断点。
    thbreak args 设置单次硬件断点。
    rbreak regex 在所有匹配regex的函数处设置断点。
    rbreak file:regex 在指定文件中设置正则断点
    info breakpoitns [list...]
    info break [list...] 打印断点信息。

    set breakpoint pending auto/on/off
    show breakpoint pending

    set breakpoint auto-hw on/off

    set breakpoint always-inserted off/on

    set breakpoint condition-evaluation host/target/auto

5.2 继续运行与单步调试

continue [ignore-count]
c [ignore-count]
fg [ignore-count]
恢复执行。当前停到的断点会被暂时忽略（停留的位置在断点前）。 要恢复到不同的执行位置，可以使用return或jump命令。
step
直到运行到不同的行。 遇到没有调试信息的函数时会被略过。 step只会停到一行代码的第一条指令上。
step count
step重复count次。
next [count]
运行到当前frame对应的函数的不同行。 next只会停留到一行代码的第一条指令上。
set step-mode
set step-mode on
set step-mode off
show step-mode
设置为on时，step指令会停留到没有调试信息的函数的第一条指令上。默认值为off。
finish
恢复运行，知道当前frame的函数返回。
set print finish [on|off]
show print finish
设置finish指令是否打印函数的返回值。
until
u
运行到当前函数的，地址更靠后的一行。例如，当在一个循环的末尾时，next会跳转到循环的开头，u会越过整个循环。 当until指令试图退出当前frame时，总会停下来。 有时机器码的顺序和源代码不同。 无参数的until通过单条指令运行的方式执行，因此会比有参数的until慢。
until location
u location
继续运行，直到达到指定的行，或者当前函数返回。 只有指定的行在当前函数帧中达到才会停止（也就是说可以用于跳过递归函数调用）。 通过设置断点的方式运行，从而速度比无参数的until快。
advance location
继续运行，指导达到指定的位置，或当前函数返回。与until不同，advance不会越过递归函数调用。
stepi
stepi arg
si
执行单条机器码指令。display /i $pc可以自动显示下一条需要执行的指令。
nexti
nexti arg
ni
执行单条机器码指令。如果是一条函数调用指令，则运行到函数返回。
set range-stepping
show range-stepping
控制range stepping的开关。默认为on。

5.3 跳过函数和文件

Posix 系统上，指定的正则表达式的格式为"Extended Regular Expression" (man 7 regex)，非posix系统上为regcomp函数接受的正则表达式。glob为"man 7 glob"。

skip [options]

选项包括

  • -file file/-fi file 文件中的函数。
  • -gfile file-glob-pattern/-gfi file-glob-pattern 匹配glob的文件中的函数。
  • -function linespec/-fu linespec 包含linespec的函数。
  • -rfunction regexp/-rfu regexp 函数名匹配regexp的函数。

无参数时，当前函数会被跳过。

skip function [linespec]
跳过包含linespec的函数。不指定function时，表示当前函数。
skip file [filename]
所有文件filename中的函数。不指定filename时，表示当前源文件。
info skip [range]
打印指定范围内的skip的信息。不指定range时，所有skip的信息都会被打印。
skip delete [range]
删除范围内的skip。不指定range时，删除所有skip。
skip enable [range]
激活range内的skip。不指定range时，激活所有skip。
skip disable [range]
disable range内的skip。不指定range时，disable所有skip。
set debug skip [on|off]
show debug skip
设置与显示是否打印skip信息。

6 反向执行

部分target支持反向执行，可以通过record或record btrace命令激活；full system emulator通常直接支持反向执行。

revrese-continue [ignore-count]
rc [ignore-count]
反向继续运行。会停留到breakpoint和synchronous exception/signal上，asynchronous signal的行为取决于目标环境。
reverse-step [count]
反向单步执行。只会停留在一行代码的开头。如果上一行代码包括函数调用，则会停留在被调用函数的最后一句（通常是return）。若函数调用不包含调试信息，则不会停留。
reverse-stepi [count]
反向执行一条机器指令。要执行的指令不是当前pc正指向的指令，而是pc之前指向的指令。
reverse-next [count]
反向执行当前函数中上一行代码。在函数的第一行，reverse-next会运行到调用函数的地方。
reverse-nexti [count]
反向执行上一条机器码。当上一条机器码是函数返回时，会执行到当前函数帧中的函数调用语句。
reverse-finish
执行到当前函数的开头，停留在开头(函数被调用之后)。
set exec-direction
set exec-direction reverse
set exec-direction forward
设置执行的方向。影响到的指令为step, stepi, next, nexti, continue, finish。return不受影响。

7 记录inferior的运行并重放

process record and replay target

replay mode

record mode

record method
method参数可以是
- full GDB's software record and replay implementation
- btrace format。只支持Intel处理器；不记录数据，从而无法查看变量和寄存器；remote debugging时，disconnect后继续record。format可以是
    - bts Branch Trace Store格式
    - pt Intel Processor Trace格式

process record and replay target中自动禁用displaced stepping；full recoding mode不支持non-stop mode和asynchronous execution mode。

record stop
停止process record and replay target。在record模式停止时，可以以非record模式继续运行；以replay模式停止时，可以以当前停止到的状态继续运行。
record goto
转到execution log指定位置。 - record goto begin/record goto start 转到execution log的开头。 - record goto end 转到execution log的结尾。 - record goto n 转到execution log的第n条指令。
record save filename
保存execution log到文件。默认文件名为gdb_record.process_id。不是所有的recording method都支持。
record restore filename
从filename中恢复execution log。文件必须是用record save创建的。
set record full insn-number-max limit
set record full insn-number-max unlimited
show record full insn-number-max
设置full recording method最大记录的指令数。默认200000。
set record full stop-at-limit
show record full stop-at-limit
控制记录的指令数到达limit时是否停下来。
set record full memory-query
show record full memory-query
控制当gdb无法记录指令造成的memory更改时的行为。
info record
查看关于记录的信息。
record delete
在replay模式下，删除后续execution log，并开始从当前地址记录行的execution log。
record instruction-history

从execution log之反汇编指令。

接受的修饰符如下:

  • /m 混合模式（源码+汇编）
  • /s 源码
  • /r raw模式
  • /p 不标记出当前指令
  • /f 不标记出当前函数

speculatively executed instrution以?标出，并不是所有的记录格式都支持这一特性。

指定反汇编的指令的范围：

  • record instruction-history insn 从insn开始的10条指令。
  • record instruction-history inst, +/-n insn后/前的n条指令。
  • record instruction-history 上次反汇编位置后的10条指令。
  • record instruction-history - 上次反汇编指令前的10条指令。
  • record instruction-history begin, end begin end范围内的指令（闭区间）。

并不是所有的recording method都支持这条指令。

set record instruction-history-size size
set record instruction-history-size unlimited
show record instruction-history-size
设置与显示record instruction-history命令默认反汇编多少条指令。默认值为10。
record function-call-history

显示函数粒度的执行历史。

修饰符:

  • /l 打印对应的行号
  • /i 打印对应指令编号
  • /c 缩进函数名以反映函数调用栈

指定要打印的函数:

  • record function-call-history func 从func开头的10个函数
  • record function-call-history func, +/-n func后/前的n个函数
  • record function-call-history func 上次打印位置后的10个函数
  • record function-call-history func - 上次打印位置前的10个函数
  • record function-call-history func begin, end begin end范围内的函数。闭区间。

并不是所有的recording method都支持这条指令。

set record function-call-history size
set record function-call-history unlimited
show record function-call-history
设置与显示record function-call-history显示行数的默认值。

9.2 指定位置

9.2.1 行位置

    linenum 当前源代码的行号。

    -offset
    +offset 相对当前行的偏移位置。对于list命令，相对于上一个打印的行;对于断点命令，想对于当前栈帧中停留的断点;当作为list的第二个参数时，想对于list的第一个参数。

    filename:linenum

    function 在C++中，会匹配所有namespace以及class中的函数。使用-qualified来匹配fully-qualified名字。

    function:label 匹配函数中的label。

    label 选择当前栈帧对应函数中的label。

    `-pstap|-probe-stap [objfile:[provider:]]name` static probe point

9.2.2 显式位置

-source filename 要区分相同basename的文件，在文件的basename前添加足够多的目录名。

-function function 在C++中，指所有作用域中名为function的函数，包括namespace和class。

-qualified 使function匹配fully-qualified name。也可用于行位置。

-label label 指定函数中的label。当未指定函数时，在所选择的栈帧中搜索label。

-line number 指定行偏移，可以是绝对(-line 3)或者相对(-line +3)。

9.2.3 地址位置

*address，其中address可以是任意表达式。

expression 当前working language的任意合法表达式。

funaddr 指定函数的第一条指令的地址，在栈帧和参数被设置之前。

'filename':funcaddr

9.5 指定源码目录

source path: gdb搜索源文件的路径

source path总包含编译目录$cdir和当前工作目录$cwd

$cwd与.的区别： .在加入source path时被展开为当前工作目录，$cwd可作为source path的一项出现，表示当前工作目录。

If a compilation directory is recorded in the debug information, and GDB has not found the source file after the first search using source path, then GDB will combine the compilation directory and the filename, and then search for the source file again using the source path.
